# UI 指南

本文档说明视觉和组件使用规则。基础代理规则见 `AGENTS.md`。交付前体验验收见 `docs/ui-quality-checklist.md`。

## 视觉方向

默认风格是 C 端、轻量 colorful、留白充足。视觉氛围可以使用柔和、多点、流动感的渐变建立品牌温度，用少量半透明玻璃材质表达悬浮层级。主体仍以清晰阅读、稳定对比和可维护组件为第一优先级。

避免：

- 后台仪表盘式密集布局。
- 大量嵌套卡片。
- 单调灰色页面。
- 泛泛的 SaaS 营销文案。
- 过度渐变和装饰图形。
- 大面积低透明玻璃态内容区。
- 把固定主色硬编码到业务组件中。

偏好：

- 清晰页面层级。
- 移动优先响应式布局。
- 大而可读的主要操作。
- 真实 C 端流程，而不是占位展示。
- 渐变作为背景氛围层，内容承载区保持高可读性。
- 玻璃态只用于导航、浮动操作、弹窗、popover 或少量强调面板。

## Mesh Gradient

Mesh Gradient 只作为视觉氛围层，不作为内容容器本身。优先用于首页首屏、CTA 区块、登录页背景、空状态插图背景或产品页局部装饰。

使用要求：

- 优先使用静态 CSS 多层 `radial-gradient()` 或静态图片背景。
- 动态渐变必须低频、克制，并尊重 `prefers-reduced-motion`。
- 渐变上方的文字必须有稳定底色、遮罩或足够对比，不依赖背景刚好可读。
- 色彩应从整体品牌氛围和主题 token 推导，不预设固定主色。

不要：

- 每个区块都使用渐变。
- 在正文、表单、长列表或商品信息区域背后使用复杂渐变。
- 为了背景动效扩大客户端交互组件范围。
- 使用模板感很强的单一紫蓝渐变作为默认视觉。

## Glassmorphism

Glassmorphism 只用于表达少量悬浮层级。它不是默认卡片样式，也不适合承载大段正文、主表单或核心商品信息。

使用要求：

- 必须有半透明背景、边框、阴影和 `backdrop-filter` 降级方案。
- 玻璃层上的正文对比度目标不低于 WCAG AA 文本要求。
- 控件边界、选中态和 focus ring 必须清晰可见。
- 移动端必须检查文字不溢出、不遮挡、不被固定导航覆盖。

建议形态：

- 顶部导航滚动后的轻玻璃态。
- Hero 区域的小型信息面板。
- 浮动操作条、popover、dialog 或移动端底部操作栏。

不要：

- 把整页、整组 pricing card、整张表单或长内容区做成玻璃态。
- 只设置透明度而不处理可读性、边界和 fallback。
- 使用多层玻璃卡片嵌套制造复杂层级。

## 主题 token

主题变量集中在 `app/assets/css/tailwind.css`。组件优先使用语义 token 和 Tailwind v4 主题变量。

常用语义：

- `background`
- `foreground`
- `primary`
- `primary-foreground`
- `secondary`
- `muted`
- `border`
- `ring`
- `destructive`

不要在业务组件中散落硬编码品牌色。确需局部强调时，优先新增语义化主题变量或项目级包装组件，不在页面中直接沉淀固定主色。

## 组件使用

- 页面布局使用 `PageContainer` 和 `PageHeading`。
- 主要按钮使用 `AppButton`。
- 空、错、加载状态使用 `EmptyState`、`ErrorState`、`LoadingState`。
- 图标优先使用 `@lucide/vue`。
- shadcn-vue 基础组件只放在 `app/components/ui`。
- 跨页面稳定业务用法放到 `app/components/common` 包装。
- 页面级复杂交互放在 `app/features/*`，页面文件保留数据编排和 SEO。

## 表单和状态

- 表单优先使用 vee-validate + Zod。
- 表单校验错误必须可见、可读。
- 异步 UI 应考虑加载、错误、空数据、禁用和成功反馈。
- 不要用客户端展示状态替代服务端权限判断。
- 用户触发刷新、局部面板和 mutation 后 refetch 可以使用 Vue Query。
- 首屏 SEO 数据不要无理由搬到客户端查询。

## 动效

动效只能出现在明确客户端交互组件中。动效应服务于状态变化或层次引导，不要让整页为了动画承担复杂浏览器状态。

要求：

- 常规微交互控制在 150-300ms。
- 优先动画 `transform` 和 `opacity`。
- 尊重 `prefers-reduced-motion`。
- 避免装饰性循环动效和导致 layout shift 的动效。

## 响应式

所有页面都必须在移动端可读。按钮文字不能溢出，卡片不能互相遮挡，首屏内容不能被固定导航覆盖。

至少检查：

- 375px 手机宽度。
- 768px 平板宽度。
- 1024px 小桌面。
- 1440px 宽屏。

## 浏览器基线

项目使用 Tailwind CSS v4，因此有效浏览器下限是：

- Chrome 111+
- Edge 111+
- Safari 16.4+
- Firefox 128+

不要只按 Vite 默认目标判断 UI 兼容性。如果业务必须支持更旧浏览器，应先讨论是否降级到 Tailwind v3.4 或建立独立兼容方案。
