# ADR：引入 Nuxt i18n

日期：2026-06-12

## 状态

已接受

## 决策负责人

项目维护者确认引入 i18n 能力。

## 背景

Colorway Starter 需要提供中英双语示例，并保持现有中文 URL 不破坏。i18n 原本不属于默认能力，按仓库规则需要人工确认和决策记录。

## 决策

引入 `@nuxtjs/i18n`，首版支持中文和英文：

- 默认语言为中文，现有路径不加语言前缀。
- 英文路径使用 `/en` 前缀。
- 禁用浏览器语言自动跳转，避免 starter 在不同浏览器环境下表现不稳定。
- 用户可见页面文案、SEO、导航、表单校验、toast、mock 商品、mock 用户资料和推荐数据都本地化。
- Vue 组件内显式使用 `useI18n()`，不使用模板实例上的 `$t`。

## 备选方案

- 仅预留 i18n 结构：依赖和维护成本低，但无法展示完整双语 starter。
- 所有语言都加路径前缀：URL 规则一致，但会破坏现有中文路径。
- 通过 cookie 或浏览器语言切换且不改 URL：实现简单，但 SEO 和可分享语言 URL 较弱。

## 影响

- 新增开发依赖 `@nuxtjs/i18n`。
- `nuxt.config.ts` 增加 i18n 模块配置。
- 用户可见 mock 数据按 `locale` 查询参数返回对应语言。
- API route 需要校验 `locale` 参数，非法值返回稳定 400。
- E2E 需要覆盖中文默认路径和英文 `/en` 路径。

## 迁移和回滚

不涉及用户数据迁移。回滚时移除 `@nuxtjs/i18n`、`i18n/locales/*`、locale 查询参数和语言切换入口，并恢复文案为单语言。

## 验证方式

- 语言包 key 一致性单测。
- API route locale 行为单测。
- `$t` 使用扫描单测。
- Playwright 覆盖中文和英文核心页面、语言切换和表单校验。
- 运行 `corepack pnpm@11.6.0 verify`、`corepack pnpm@11.6.0 test:e2e` 和 Docker smoke。
